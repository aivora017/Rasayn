P0: 2. P1: 8. P2: 6. P3: 5. Total items: 21.

Scope: `apps/desktop` (TS + Rust/Tauri) and `packages/*`. Read-only audit; no

Headline findings:

## P0 — ship-blockers before GA

### D01 `apply_migrations` manual-const gauntlet (every new migration = 2 edits)
**Location**: `apps/desktop/src-tauri/src/db.rs:28-207`
**Category**: arch
**Cost now**: Every new SQL file under `packages/shared-db/migrations/`
requires (a) a new `pub const MIGRATION_NNNN: &str = include_str!(...)` and
(b) a new `if !applied(N, conn) { ... }` stanza. There are currently 18
migrations and the `applied(...)` block is 126 lines of copy-paste. The
next X1.3 HSN plumbing migration will be the 19th and will re-hit this
friction. Memory note (2026-04-15 audit) flagged it as P1; now it is the
single biggest recurring-cost smell in the repo.
**Cost at scale**: At 100 shops this does not scale (data is identical), but
every developer cutting a release branch is taxed ~5 minutes + risks
off-by-one typos in the version number / name string. A missed `applied(N)`
branch = silent no-op migration in the field.
**Fix sketch**: Use `rust-embed` or a `build.rs` glob to enumerate
`migrations/*.sql`, sort by the leading 4-digit number, loop. Pattern:
```rust
for (version, name, sql) in MIGRATIONS.iter() { ... }
```
Drop the 18 hand-written constants and 18 if-branches down to ~15 lines.
**Effort**: S (half a day, including a regression test that replays a v0 DB
forward).

### D02 `DbState` tuple-struct with `.0.lock()` spread everywhere
**Location**: `apps/desktop/src-tauri/src/db.rs:8` (definition);
`commands.rs`, `products.rs`, `products_perf.rs`, `images.rs`, `oauth/mod.rs`
(63 call sites matching `state.0.lock()`).
**Category**: arch
**Cost now**: Every command handler opens with
`let conn = state.0.lock().map_err(...)?;`. The `.0` is magic indexing into
the unnamed `Mutex<Connection>` field; new contributors grep for `.lock()`
and hit 63 identical blocks. Error-mapping boilerplate is copy-pasted.
**Cost at scale**: Every new command (A14+, next sprints' A8 deferred work,
X1.3 HSN plumbing) repeats 3-4 lines of the same locking preamble.
Refactoring the mutex (e.g. to `parking_lot::Mutex`, or to a per-request
connection pool when we need concurrent reads) touches 63 sites.
**Fix sketch**: Rename field to `conn`
(`pub struct DbState { pub conn: Mutex<Connection> }`) and add a helper
`DbState::with<T>(&self, f: impl FnOnce(&Connection) -> Result<T>) -> Result<T>`
that handles the poison-error mapping once. Call sites become
`state.with(|c| { ... })`.
**Effort**: M (1 day; mechanical but touches every command file).

## P1 — backlog, address in next 2 sprints

### D03 `SUPPLIERS` demo list hardcoded in GrnScreen
**Location**: `apps/desktop/src/components/GrnScreen.tsx:19`
(`const SUPPLIERS: readonly { id: string; name: string }[] = [...]`).
**Category**: dead-code / arch
**Cost now**: GRN save uses a fake supplier id drawn from a frontend
constant instead of `list_suppliers` IPC. `list_suppliers` is defined in
`ipc.ts:98` but the screen never calls it. Demo data ships in prod bundle.
**Cost at scale**: Every new shop sees identical fake supplier names. GRN
saves may reference supplier IDs that don't exist in `suppliers` table on
that shop — the FK on `grns.supplier_id` (migration 0003) will reject them.
**Fix sketch**: Replace `SUPPLIERS` const with a `useEffect` that calls
`listSuppliersRpc(shopId)`. Add loading state; fall back to a "Create
supplier" inline action if the list is empty (fresh shops).
**Effort**: S.

### D04 Non-FEFO-critical migrations missing FK covering indexes
**Location**: `packages/shared-db/migrations/0005_oauth_and_audit.sql`
(oauth_accounts.shop_id, no index); `0004_supplier_templates.sql`
(supplier_id REFERENCES but `idx_supplier_templates_shop` only covers
shop_id); `0017_x2_product_images.sql` (uploaded_by FK unindexed);
`0011_expiry_override_audit.sql` (product_id FK unindexed —
only batch_id composite is indexed).
**Category**: schema
**Cost now**: Negligible in single-user pilot.
**Cost at scale**: At 100 shops × 3 years of audit data, queries like
"all overrides by pharmacist X" (actor_user_id) or "all images uploaded by
user Y" do a full-scan. Audit CSV exports will slow quadratically.
**Fix sketch**: Add a `0019_fk_indexes.sql` migration with 4-5 covering
indexes on the named FK columns.
**Effort**: S.

### D05 `commands.rs` monolith (4342 lines)
**Location**: `apps/desktop/src-tauri/src/commands.rs`
**Category**: arch
**Cost now**: Every new command pushes this higher. Rust compile times on
the desktop crate are dominated by this file. Merge conflicts during
parallel A-card work almost always land here.
**Cost at scale**: Becomes unreviewable. New contributors can't find handlers.
**Fix sketch**: Split by domain following the IPC groupings already in
`ipc.ts`: `commands/billing.rs`, `commands/grn.rs`, `commands/reports.rs`,
`commands/directory.rs`, `commands/einvoice.rs`, `commands/reconcile.rs`,
`commands/product_master.rs`, `commands/images.rs` (the last already exists
as a top-level module). Re-export from `commands/mod.rs`.
**Effort**: M.

### D06 `ipc.ts` monolith (1119 lines)
**Location**: `apps/desktop/src/lib/ipc.ts`
**Category**: arch
**Cost now**: DTO interfaces, IpcCall union, RPC wrappers all live in one
file. Touching any one forces CI to re-typecheck the lot. Code navigation
is painful (16 domain sections glued together with comment banners).
**Cost at scale**: Same re-checking cost per PR; pain grows with each new
A-card.
**Fix sketch**: Split into `lib/ipc/types/*.ts` per domain (billing.ts,
grn.ts, reports.ts, etc.) and a thin `lib/ipc/index.ts` that keeps the
IpcCall union assembled and exports the RPC functions. Tree-shaking
improves too.
**Effort**: M.

### D07 `App.test.tsx` fat test file (845 lines, ~18 Alt+digit-nav cases)
**Location**: `apps/desktop/src/App.test.tsx`
**Category**: test
**Cost now**: Runs every test suite; a single navigation regression flakes
the whole file. Hard to find the specific scenario causing a failure.
**Cost at scale**: Alt+digit slots are exhausted (per memory note). Next
nav key added will mean editing this file again.
**Fix sketch**: Split per destination screen under `apps/desktop/src/nav/`
(e.g. `nav-to-billing.test.tsx`, `nav-to-inventory.test.tsx`). Extract
`fireAltDigit(n)` helper into `src/test/navHelpers.ts`.
**Effort**: S.

### D08 `sku-images` package dep-version drift
**Location**: `packages/sku-images/package.json:20-21`
(`typescript ^5.4.0`, `vitest ^1.6.0`).
**Category**: deps
**Cost now**: Rest of the monorepo is on `typescript ^5.6.3` and
`vitest ^2.1.5`. Tests in this package run against an older Vitest; any
Vitest 2.x-only matcher used accidentally in shared helpers would fail
here first.
**Cost at scale**: Blocks the eventual `npm-check-updates` sweep; creates
subtle "works on my machine" moments when CI's hoisted versions differ
from the package-level pin.
**Fix sketch**: One-line bump to match the other packages. Re-run `npm
install` and vitest.
**Effort**: S.

### D09 Perf-json churn requires manual `git checkout` pre-commit
**Location**: `docs/evidence/a{1..6,11}/perf.json`
**Category**: arch (tooling)
**Cost now**: Per memory note, every local perf-test run rewrites these
files with host-specific timings; contributors must `git checkout
docs/evidence/` before `git add`. Missed once = noisy PR diff;
missed repeatedly = real regressions get hidden in the noise.
**Cost at scale**: On every contributor's machine, not just one. Every
CI run overwrites them too.
**Fix sketch**: Either (a) move perf.json to a `.gitignore`d cache and
upload as a CI artifact; or (b) add a pre-commit hook that restores
`docs/evidence/**/perf.json` unless `PHARMACARE_PERF_UPDATE=1`. Option (a)
is cleaner; ADR pending.
**Effort**: S.

### D10 `BillingScreen.tsx` bloat (1037 lines, multiple concerns)
**Location**: `apps/desktop/src/components/BillingScreen.tsx`
**Category**: arch
**Cost now**: One component owns: line entry, RX modal gating, keyboard
shortcuts (F-keys + Alt guards), expiry-threshold UI, payment triggering,
amount rounding display. Cognitive load is high.
**Cost at scale**: Every A-card that touches billing (A14 returns, future
loyalty, future split-tender UI) edits this file and risks breaking
unrelated keyboard behaviour.
**Fix sketch**: Extract `useBillKeyboard()` hook, `<BillLineEditor/>`,
`<ExpiryBadge/>` subcomponent, `<TotalsPanel/>` subcomponent. Keep
`BillingScreen` as the coordinator.
**Effort**: M.

## P2 — nice to have

### D11 Startup `.expect()` calls could degrade gracefully
**Location**: `apps/desktop/src-tauri/src/main.rs:27,28,98`;
`oauth/loopback.rs:121`; `oauth/gmail_api.rs:52`.
**Category**: arch
**Cost now**: App panics with a cryptic message if the data dir is
read-only or migration files are corrupt. The `gmail_api.rs:52` reqwest
client build is particularly dangerous — it panics inside what the caller
expects to be a fallible async function.
**Fix sketch**: Convert to `?` with a user-visible splash error.
**Effort**: S.

### D12 MIME-sniff duplicated client+server (by design, but unify list)
**Location**: `packages/sku-images/src/mime.ts` and
`apps/desktop/src-tauri/src/images.rs:31-45`.
**Category**: duplication
**Cost now**: Two copies of PNG/JPEG/WebP magic-byte tables. If WebM or
AVIF is ever added, both sides must be edited in sync. Silent drift would
let the frontend accept an AVIF the server rejects.
**Fix sketch**: Treat sku-images as the single source; generate a small
Rust match table from it via a `build.rs` or commit-hook-checked script.
Or just add a unit-test in each side that asserts the list length matches
an agreed constant.
**Effort**: S.

### D13 Default-shop / default-user seed lives in db.rs, not seed-tool
**Location**: `apps/desktop/src-tauri/src/db.rs:213-249`
(`ensure_default_shop`, `ensure_default_user`).
**Category**: arch
**Cost now**: Seed values (e.g. `'00AAAAA0000A0Z0'` GSTIN) are compiled
into the Rust binary. Changing them requires a native rebuild, not just a
TS deploy. Setup docs aren't co-located with data.
**Fix sketch**: Move the seeds into `packages/seed-tool` as a "bootstrap"
step run once on fresh-install. Keep the idempotent guard where it is but
have it INSERT from a TS-provided fixture.
**Effort**: M.

### D14 `phash.rs` test unwraps galore (20+) — acceptable but could use helper
**Location**: `apps/desktop/src-tauri/src/phash.rs:156-242`.
**Category**: test
**Cost now**: Readability only. Noise.
**Fix sketch**: Add `fn hash(bytes: &[u8]) -> Hash { compute_phash(bytes).expect("phash on valid fixture") }` to the test module and reuse it.
**Effort**: S.

### D15 Bills `round_off_paise BETWEEN -50 AND 50` is the only CHECK on rounding
**Location**: `packages/shared-db/migrations/0001_init.sql:142-143`.
**Category**: schema
**Cost now**: `grand_total_paise % 100 = 0` is enforced, but the invariant
`subtotal - discount + cgst+sgst+igst+cess + round_off = grand_total` is
NOT checked by SQL. Application-layer bug could write an impossible total.
**Fix sketch**: Add a CHECK at insert time (requires either a trigger or
the arithmetic in a CHECK that SQLite accepts).
**Effort**: M (needs GST-intra vs inter handling — be careful).

### D16 Optional columns that should probably be NOT NULL
**Location**: `packages/shared-db/migrations/0001_init.sql`: `customers.phone`,
`suppliers.gstin`, `suppliers.phone`, `doctors.phone`.
**Category**: schema
**Cost now**: Nulls make search/indexing less predictable (phone index
has NULL rows). DirectoryScreen has to handle nulls everywhere.
**Fix sketch**: For customers without phone, store empty string + a CHECK
that at least one of {phone, address} is set. Low priority; document &
defer if pilots disagree.
**Effort**: M.
